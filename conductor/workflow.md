# Workflow

## Cobertura de Tests

- Se requiere >80% de cobertura de código.

## Commits

- Commit después de cada tarea.
- Los commits los realiza el desarrollador manualmente.
- Mensajes de commit en español.
- Formato convencional: `tipo: mensaje descriptivo`.

## Registro de Tareas

- Usar `git notes` para agregar el resumen de cada tarea completada.

## Estructura de Tareas

- Cada tarea debe seguir el flujo: análisis → implementación → test → commit.
